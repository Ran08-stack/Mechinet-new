## איפה אנחנו בפועל (תמצית מהירה אם תרצה לעיין בעצמך)

### מה נבנה ועובד
- Auth + magic link
- Form Builder + טופס הגשה ציבורי `/apply/[formId]`
- דשבורד מועמדים — חיפוש, פילטר שלב, bulk actions, רישום אירועים
- פרופיל מועמד — פרטים, קבצים, AI summary, הערות, היסטוריית פעילות
- לוח בקרה — KPIs, משפך שלבים אינטראקטיבי, מועמדים אחרונים, LiveActivity
- הגדרות מכינה — שם, לוגו, ניהול שלבי pipeline דינמיים, סוג מכינה
- ארכיטקטורת שני חשבונות: council_admin (`/council`) + org_admin (`/dashboard`)
- שלבי pipeline דינמיים per organization (לא יותר 5 שלבים אנגליים קשיחים)
- candidate_events — לוג מלא של כל פעולה (stage_changed, note_added, ai_summary, form_submitted, interview_scheduled)
- 16 מסכי עיצוב מוכנים ב-`mechinet - ui/` (Navy/Orange/Teal, Rubik, RTL)

### מה לא נבנה / נדחה
- פורטל מועמד (מסך 10) — הקוד מוכן אבל חסום ב-RLS, דורש סוג משתמש שלישי
- ציטוטים על AI summary — נדחה
- WhatsApp / SMS — מחוץ ל-MVP
- AI שמקבל החלטות אוטומטיות — לא רוצים, AI עוזר בלבד
- שלב עיצוב פיקסל-בפיקסל לפי המוקאפים — נדחה לסוף

### מה השלב הבא (לפי הצ'אט האחרון)
- אחרי deploy של event logging מלא: לבדוק שכל סוגי האירועים מופיעים ב-LiveActivity
- כשמגיעים למסך יומן/ראיונות — לוודא ש-interview_scheduled מסתנכרן ל-LiveActivity
- אחר כך: עיצוב מדויק לפי המוקאפים

## כללים שחשוב שיעבור איתי ל-Claude Code

### תקשורת
- עברית בלבד.
- בלי "בואו נצלול", בלי "חשוב לציין", בלי "מעולה!".
- בלי אימוג'ים.
- Bullets ולא פרוזה ארוכה.
- לא לערבב עברית ואנגלית באותה שורה (קשה לי לקרוא).

### כתיבת קוד
- פשטות. מינימום קוד שפותר את הבעיה.
- כירורגיה. נגע רק במה שנדרש. אל תשפר קוד סמוך. אל תמחק dead code שלא שלך.
- 200 שורות מקס' לתוצר — אם אפשר 50, כתוב 50.
- הצהר הנחות לפני שמתחיל.
- עצור כשלא ברור. שאל.

### תהליך
- לפני task מרובה שלבים — תוכנית קצרה + verify לכל שלב.
- אחרי כל החלטה / תוצאה / בלוקר → append ל-`memory/memory.md` בלי לבקש אישור.
- שינוי כיוון / פיצ'ר חדש → הצעה ל-`memory/context.md` + לחכות לאישור.
- כלל חדש → הצעה ל-`CLAUDE.md` + לחכות לאישור.

### Deploy
- לעולם לא `git push` בפרויקט הזה.
- רק `vercel --prod` מהטרמינל.
- לפני deploy: `npx tsc --noEmit` + `npm run build` כדי לוודא.

### Supabase
- כל שינוי schema → migration + RLS policy + רענון `types/database.ts`.
- יש Skill בשם `mechinet-supabase-migration` שמתעד את התהליך.
