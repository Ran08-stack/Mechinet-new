# עיצוב — מבנה מכינות/שלוחות + מפת מועצה

תאריך: 2026-05-28
מצב: טיוטה לבדיקת רן

## רקע ומטרה

לוח המועצה צריך מפה אינטראקטיבית של המכינות בארץ. בבירור התברר שזה לא "פיצ'ר מפה"
אלא שינוי מודל נתונים: כדי שמפה תהיה אמיתית, צריך שלכל **שלוחה** יהיה חשבון משלה,
שתשתייך ל**מכינה** (קטגוריה שהמועצה יוצרת), ושיהיה לה **מיקום**.

החלטות מהבירור עם רן:
- **אופציה 1 נבחרה:** כל שלוחה = חשבון נפרד שמנהל את המועמדים שלו. ה"מכינה" היא קטגוריה מקבצת.
- שלוחה מופיעה על המפה רק אחרי שהמועצה הוסיפה אותה.
- פרטי מיקום נראים רק למועצה.
- את פירוט שדות ה-onboarding (כולל רמת דיוק המיקום) נחליט בשלב 2.

## המבנה ההיררכי

```
תנועה (movements, קיים — אופציונלי)
   └── מכינה (academies — חדש, נוצר ע"י המועצה)
          └── שלוחה (organizations — קיים; מקבל academy_id + מיקום בשלב 2)
                 └── מועמדים (candidates — קיים)
```

דוגמאות אמיתיות:
- תנועת החלוץ → מכינת רבין (שנתית, 4 שלוחות) + מכינת רעות (חצי-שנתית, 4 שלוחות)
- מכינת ערבה (שנתית, 4 שלוחות) — ללא תנועה

## מצב נתונים קיים (נבדק ב-DB)

- `movements` קיים.
- `organizations` כבר כולל: `movement_id`, `branch_name`, `region`, `contact_person`, `contact_phone`, `status`.
  כלומר כל org היום הוא בפועל **שלוחה** (name="מכינת רעות", branch_name="שלוחת אורנים").
- **חסרה טבלת `academies`** — שכבת הקיבוץ האמצעית.
- רשומות קיימות: רעות (archived, movement משויך), נחשון (active, ללא movement).

## פירוק ל-3 תתי-פרויקטים (כל אחד spec+plan נפרד)

1. **מודל הנתונים** ← השלב הזה. טבלת `academies` + שיוך השלוחות.
2. **Onboarding + ניהול** — המועצה יוצרת מכינה, מוסיפה שלוחה עם פרטים+מיקום; סינון לפי מכינה במסך הרשימה.
3. **המפה** — מציגה שלוחות אמיתיות לפי מיקום, לחיצה→drill למכינה/שלוחה.

תנועה (שכבה רביעית) כבר קיימת חלקית; נחבר אותה דרך `academies.movement_id`.

## שלב 1 — מודל הנתונים (הסקופ של ה-spec הזה)

### טבלה חדשה: `academies` (מכינה)

| עמודה | טיפוס | הערות |
|---|---|---|
| id | uuid PK | default gen_random_uuid() |
| name | text NOT NULL | שם המכינה, למשל "מכינת רבין" |
| movement_id | uuid NULL | FK→movements(id), אופציונלי |
| program_type | text NULL | 'annual' \| 'half_year' (שנתית/חצי-שנתית) — אופציונלי |
| created_at | timestamptz | default now() |

### שינוי ב-`organizations` (שלוחה)

- הוספת עמודה `academy_id uuid NULL REFERENCES academies(id)`.
- `organizations.movement_id` הופך מיותר (התנועה נגזרת דרך המכינה). **לא מוחקים** כדי לא לשבור קוד קיים — מסמנים כ-deprecated ונקה בשלב מאוחר.
- שדות מיקום (עיר/קואורדינטות) — **נדחים לשלב 2** לפי בקשת רן.

### RLS

- `academies`: `council_admin` (דרך `is_council_admin()`) — קריאה+כתיבה מלאה.
- חבר מכינה (org member) — קריאה בלבד של המכינה שאליה השלוחה שלו משויכת.

### Backfill / seed

- ליצור `academies` לשתי הרשומות הקיימות ולשייך:
  - "מכינת רעות" (movement של רעות) ← org reut-oranim.
  - "מכינת נחשון" ← org נחשון.
- כך כל org קיים מקבל academy_id, האפליקציה נשארת עקבית.

### עדכון קוד

- `types/database.ts`: הוספת טיפוס `Academy` + `academy_id` ל-`organizations`.
- **אין שינוי UI בשלב הזה.** המסכים הקיימים ממשיכים לעבוד (academy_id nullable, לא נדרש בשום query קיים).

### Verify

- `tsc` נקי.
- query בדיקה: כל org קיים בעל `academy_id` לא-null אחרי ה-seed.
- צד מכינה וצד מועצה נטענים כרגיל (לא נשבר כלום).
